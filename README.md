# Concept Check

## 1. What is the difference between WHERE and HAVING?

- **WHERE** filters rows before grouping or aggregation takes place.
- **HAVING** filters groups after the GROUP BY clause has been applied.

Use **WHERE** to filter individual records and **HAVING** to filter aggregated results.

---

## 2. When would you use a correlated subquery instead of a JOIN?

A correlated subquery is used when the inner query depends on values from the current row of the outer query. It executes once for each row returned by the outer query.

It is useful when comparing each row against related records, such as finding the highest rental rate within each film category.

---

## 3. What is a CTE, and why is it more readable than a nested subquery?

A **Common Table Expression (CTE)** is a temporary result set created using the `WITH` keyword.

CTEs improve readability because they:

- Break complex problems into smaller logical steps.
- Reduce deeply nested queries.
- Improve maintainability.
- Make debugging easier.

---

## 4. Explain the difference between RANK() and DENSE_RANK().

Both functions assign rankings to rows.

### RANK()

If two rows share the same rank, the next rank is skipped.

Example:

```
Score
100
100
90

Ranks
1
1
3
```

### DENSE_RANK()

If two rows share the same rank, the next rank is not skipped.

```
Score
100
100
90

Ranks
1
1
2
```

---

## 5. What does PARTITION BY do differently from GROUP BY?

**GROUP BY**

- Combines rows into one row per group.

**PARTITION BY**

- Divides rows into groups for window functions while keeping every original row visible.

---

## 6. Can a subquery return multiple rows? What operator would you use in that case?

Yes.

When a subquery returns multiple rows, operators such as:

- IN
- ANY
- ALL
- EXISTS

can be used depending on the requirement.

---

## 7. Give an example of when CASE WHEN is useful inside an aggregate function.

`CASE WHEN` is useful for conditional aggregation.

Example:

- Calculate revenue only from payments greater than \$5.
- Count only active customers.
- Calculate sales from a specific category.

It allows conditions to be applied inside aggregate functions such as SUM() or COUNT().

# When to Use a Subquery vs a CTE vs a Window Function

## Subquery

A subquery is used when one query depends on the result of another query.

Typical uses include:

- Comparing values against averages
- Finding maximum or minimum values
- Checking whether records exist
- Filtering results based on another query

Example:

Finding customers whose spending is greater than the average customer spending.

---

## Common Table Expression (CTE)

A CTE is used when solving a problem in multiple logical steps.

CTEs are preferred when:

- The query becomes difficult to read.
- Intermediate calculations need to be reused.
- Complex reports require multiple stages.

Example:

Calculating customer spending first and then ranking customers within each city.

---

## Window Functions

Window functions perform calculations across related rows without reducing the number of rows returned.

They are useful for:

- Ranking
- Running totals
- Previous and next values
- Row numbering
- Comparing rows

Example:

Finding the most recent rental for every customer using ROW_NUMBER().
